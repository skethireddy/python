Bug ID 420368: WAM::disable causes tmm abort if a request has an Authorization header.

Published: 2013-05-01 21:14:32  Last Modified: 2018-02-26 10:17:18

Bug Tracker

Product and Affected Module:
BIG-IP WAM

Known Affected Versions:
11.3.0, 11.2.1, 11.2.0

Fixed In:
None

Opened: 2013-05-01 21:14:32
Severity: 3-Major
Related Known Issue Article:
Vulnerability Severity:
CVE:
Related Security Advisory Article:

Symptoms

Any request (for example, WAM::disable) that occurs in an iRule event after WAM:SMART causes a RST if there is an Authorization header present.

Impact

The system posts the message: Internal error ((WAM::SMART) tmi abort failed and resets the connection.

Conditions

This occurs on WebAccelerator Manager (WAM) when any request in a TCP stream contains an Authorization header, and encounters an iRule containing an event after WAM:SMART.

Workaround

None.

Fix Information

Behavior Change
